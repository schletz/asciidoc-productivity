Aufgabe 1: Abfragen und Servicemethoden (14 Punkte)

Im Projekt Aufgabe1 befindet sich die Klasse QuoteService. Alle folgenden Implementierungen finden in dieser Klasse statt.

Es sind folgende Abfragen und Servicemethoden zu implementieren:

List<QuoteDto> GetQuotes(string ticker, int exchangeId)

Laden Sie alle Kurse für ein bestimmtes Anlagegut (anhand des ticker) an einer bestimmten Börse (exchangeId). Die Liste soll absteigend nach Datum (neueste zuerst) sortiert sein. Als Rückgabetyp steht Ihnen folgender Record zur Verfügung:

public record QuoteDto(DateOnly Date, decimal ClosePrice);

Hinweis: Verwenden Sie OrderByDescending(q ⇒ q.Date) (C#) bzw. sorted(Comparator.comparing(DailyQuote::getDate).reversed()) (Java), um die Daten zu sortieren.

ExchangeStatsDto GetExchangeStats(int exchangeId)

Laden Sie die Daten einer bestimmten Börse und ermitteln Sie zusätzlich, wie viele Kurseinteinträge (DailyQuote) für diese Börse insgesamt in der Datenbank gespeichert sind. Als Rückgabetyp steht Ihnen folgender Record zur Verfügung:

public record ExchangeStatsDto(string Name, string Location, int TotalQuotesCount);

void AddQuote(int exchangeId, string ticker, DateOnly date, decimal price)

Diese Methode soll einen neuen Datensatz in DailyQuote erstellen. Dabei sind folgende einfache Randbedingungen zu prüfen:

  • Wird die übergebene exchangeId nicht gefunden, werfen Sie mit throw new QuoteServiceException("Invalid exchange.") eine Exception.

  • Wird der übergebene ticker (z.B. "AAPL") nicht gefunden, werfen Sie mit throw new QuoteServiceException("Invalid asset ticker.") eine Exception.

  • Der Preis darf nicht negativ sein. Ist price kleiner als 0, werfen Sie mit throw new QuoteServiceException("Price cannot be negative.") eine Exception.

  • Es darf pro Tag, Anlagegut und Börse nur einen Kurs geben. Prüfen Sie, ob für das gegebene Datum, diese Börse und dieses Anlagegut bereits ein Eintrag existiert. Falls ja, werfen Sie mit throw new QuoteServiceException("Quote already exists for this date.") eine Exception.

Sind alle Bedingungen erfüllt, speichern Sie den neuen DailyQuote-Datensatz in der Datenbank.

Aufgabe 2a: Erstellen einer RESTful API (17 Punkte)

Im Projekt Aufgabe2 steht Ihnen eine Klasse ExchangesController zur Verfügung. Diese Klasse bekommt über Dependency Injection ein vorgegebenes Service (ExchangeService). Verwenden Sie dieses vorgegebene Service; Sie müssen nicht Ihren Service von Aufgabe 1 verwenden.

Das Service hat 3 Methoden bzw. Properties:

IQueryable<DailyQuote> Quotes bzw. List<DailyQuote> getQuotes()

Liefert eine Liste aller in der Datenbank befindlichen Kurse (DailyQuote).

Task<DailyQuote> AddQuote(AddQuoteCmd cmd) bzw. DailyQuote addQuote(…​)

Legt ein DailyQuote auf Basis des übergebenen Commands an und gibt dieses zurück. Wirft ExchangeNotFoundException, wenn die Börse nicht gefunden wurde. Wirft QuoteException, wenn die Daten nicht eingefügt werden können.

Task UpdateLocation(UpdateLocationCmd cmd) bzw. void updateLocation(…​)

Aktualisiert den Standort (Location) der Börse auf Basis des übergebenen Commands. Wirft ExchangeNotFoundException, wenn die Börse nicht gefunden wurde. Wirft QuoteException, wenn die Daten nicht aktualisiert werden können.

GET /api/exchanges/{exchangeId}/quotes/{quoteId}

Dieser Endpunkt liefert ein einzelnes Kurs-Objekt mit der angegebenen Id in DailyQuote.Id zurück. Filtern Sie auch nach der ID der Börse (exchangeId). Nutzen Sie hierfür im vorgegebenen Service das Property Quotes. Als Rückgabetyp steht Ihnen folgender Record zur Verfügung:

public record DailyQuoteDto(int Id, string Ticker, DateOnly Date, decimal ClosePrice);

Wird der Eintrag nicht gefunden, ist HTTP 404 Not Found mit RFC Problem Detail zurückzugeben.

POST /api/exchanges/{exchangeId}/quotes

Dieser Endpunkt trägt für eine bestimmte Börse (exchangeId) einen neuen Kurs ein. Nutzen Sie hierfür im vorgegebenen Service die Methode AddQuote. Als Command-Object steht Ihnen folgender Record zur Verfügung, den Sie mit Validierungs-Attributen ergänzen müssen:

public record AddQuoteCmd(DateOnly Date, decimal ClosePrice, int AssetId, int ExchangeId)

Achten Sie auf folgende Validierungen des Command-Objekts bzw. des Controllers:

  • Der Wert in ExchangeId muss mit dem Routing-Parameter (exchangeId) übereinstimmen.

  • Der Wert in ClosePrice darf nicht negativ sein (Minimum 0).

  • Der Wert in AssetId muss mindestens 1 sein.

  • Der Wert in ExchangeId muss mindestens 1 sein.

Wird eine dieser Bedingungen verletzt, geben Sie HTTP 400 (Bad Request) samt RFC Problem Detail zurück. Wurde die Börse nicht gefunden (AddQuote wirft eine ExchangeNotFoundException), geben Sie HTTP 404 (not found) samt RFC Problem Detail zurück. Wurde der Datensatz korrekt angelegt, geben Sie HTTP 201 (Created) mit korrektem Location-Header auf den entsprechenden GET-Endpunkt /api/exchanges/{exchangeId}/quotes/{quoteId} zurück.

PATCH /api/exchanges/{exchangeId}/location

Ändert den Standort (Location) einer Börse. Als Command-Object steht Ihnen folgender Record zur Verfügung, den Sie mit Validierungen ergänzen müssen:

public record UpdateLocationCmd(int ExchangeId, string Location);

Achten Sie auf folgende Validierungen des Command-Objekts bzw. des Controllers:

  • Der Wert in ExchangeId muss mit dem Routing-Parameter übereinstimmen.

  • Der Wert in ExchangeId muss mindestens 1 sein.

  • Der String in Location muss zwischen 1 und 255 Stellen lang sein.

Wird eine dieser Bedingungen verletzt, geben Sie HTTP 400 (Bad Request) samt RFC Problem Detail zurück. Wird die Börse nicht in der Datenbank gefunden, geben Sie HTTP 404 (Not Found) samt RFC Problem Detail zurück. Wurde der Datensatz korrekt aktualisiert, geben Sie HTTP 204 (No Content) zurück.

Aufgabe 2b: Erstellen von End-to-End (Integration) Tests (7 Punkte)

Verfassen Sie im Projekt Aufgabe2.Test in der Klasse ExchangesControllerTests Integration-Tests, die den Endpunkt POST /api/exchanges/{exchangeId}/quotes testen. Es sind keine anderen Endpunkte zu testen.

Erstellen Sie Tests für die folgenden Szenarien:

  • POST /api/exchanges/{exchangeId}/quotes kann mit gültigen Daten den Datensatz anlegen und liefert HTTP 201 Created.

  • POST /api/exchanges/{exchangeId}/quotes kann den Datensatz nicht anlegen, da der Routingparameter exchangeId nicht mit dem Wert im Command-Objekt übereinstimmt (HTTP 400 Bad Request).

  • POST /api/exchanges/{exchangeId}/quotes kann wegen eines ungültigen (negativen) Werts in ClosePrice den Datensatz nicht anlegen (HTTP 400 Bad Request).

  • POST /api/exchanges/{exchangeId}/quotes kann wegen eines ungültigen Werts in AssetId den Datensatz nicht anlegen (HTTP 400 Bad Request).

  • POST /api/exchanges/{exchangeId}/quotes kann wegen eines ungültigen Werts in ExchangeId den Datensatz nicht anlegen (HTTP 400 Bad Request).

Prüfen Sie bei allen Tests, ob der korrekte HTTP Statuscode geliefert wird.
